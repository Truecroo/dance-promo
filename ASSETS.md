# Dance Promo — полный пак ассетов

Спецификация всех ассетов для игры. Формат, размеры, нейминг, кол-во кадров
и куда это подключается в коде. Всё с прозрачным фоном (PNG, если не указано иное),
sRGB, ретина-ready (×2).

Базовая система координат игрового поля: квадрат **480×480** логических px
(`CONFIG.canvas.size`). Реальный рендер масштабируется под экран.
Ориентир арт-стиля: **чистый минимал, тёмный танцпол, один акцент (#ff4d7d)**.

## 1. Танцор 🔴

Главный ассет. 6 настроений, переключаются по уровню комбо (см. `CONFIG.moods`).

### Состояния
| mood | Когда (комбо) | Характер движения |
|------|---------------|-------------------|
| `idle`   | 0      | стоит, чуть покачивается, скучает |
| `warm`   | 1–7    | начал двигаться, лёгкий тёп-тёп |
| `groove` | 8–19   | уверенный танец, корпус в ритме |
| `fire`   | 20–39  | зажигает, активные движения рук/ног |
| `star`   | 40+    | звезда вечера, эффектные па, вращения |
| `miss`   | при промахе (вспышка ~0.35с) | споткнулся / сбился |

### Формат: спрайт-лист (атлас) — **рекомендуется**
- **6 PNG-атласов**, по одному на mood: `dancer_idle.png`, `dancer_warm.png`,
  `dancer_groove.png`, `dancer_fire.png`, `dancer_star.png`, `dancer_miss.png`
- Сетка кадров: **горизонтальный ряд**, кадры одинакового размера
- Кадров на цикл: `idle` 4–6, `warm` 6, `groove` 8, `fire` 8, `star` 10–12, `miss` 4
- Размер кадра: **256×256** (×2 = 512×512), персонаж занимает ~70% высоты,
  стоит на «полу» по нижней трети кадра, центрирован по X
- Прозрачный фон, отступы по краям 10–15% (чтобы свечение/тень не обрезались)
- Цикл должен бесшовно зацикливаться (первый и последний кадр стыкуются)

> Альтернатива 1 — **Lottie**: 6 файлов `.json` (те же состояния). Легче и резче.
> Альтернатива 2 — **минимум**: по 1 статичной позе на mood (6 PNG), анимацию доберём CSS.

### Доп. требования
- Единый персонаж во всех состояниях (один силуэт/одежда)
- Стиль читается на тёмном фоне; допустим контурный свет/ободок
- На `star` приветствуется внутреннее свечение/искры

### Подключение
Сейчас DOM-элемент `#dancer` с эмодзи. Под спрайт-лист заменю на canvas-слой
или CSS `steps()`-анимацию; в `CONFIG.dancerSprite` пропишутся пути к атласам +
число кадров.

## 2. Стрелки 🔴

Летят с 4 сторон к центру. В коде остриё всегда направлено к центру.

### Вариант A (минимум) — 1 спрайт
- `arrow.png` — одна стрелка остриём **вверх**, **128×128** (×2 = 256)
- Код поворачивает её под каждое направление. Достаточно для старта.

### Вариант B (рекомендуется) — по цвету направления
4 файла, цвета из `CONFIG.dirColors`:
- `arrow_up.png` (#ff5a8a), `arrow_down.png` (#4ad6ff),
  `arrow_left.png` (#ffd23f), `arrow_right.png` (#7c5cff)
- **128×128**, остриё к центру (up→вверх, down→вниз, left→влево, right→вправо)

### Опционально — состояние «hit/perfect»
- `arrow_<dir>_glow.png` ×4 — подсвеченная версия (вспышка в момент попадания)

### Доп.
- Чёткий силуэт, толстая заливка, лёгкое внешнее свечение
- Можно SVG вместо PNG (резче на масштабе)

---

## 3. Кольцо-приёмник и зона попадания 🟡

Центральная мишень (сейчас рисуется кодом, `CONFIG.target.radius = 74`).

- `target_ring.png` — кольцо-приёмник, **внешний Ø ~200** (под радиус 74×2 + запас),
  тонкая обводка, нейтральный полупрозрачный
- `target_pulse.png` — кольцо для анимации пульса при попадании (можно тот же,
  код масштабирует и гасит прозрачность). Если хочешь кастом — отдельный файл
- (опц.) `receiver_<dir>.png` ×4 — «гнёзда» приёмника по сторонам, в которые
  стрелка влетает (контурные версии стрелок)

---

## 4. Фон сцены 🟡

Вертикальная ориентация (мобайл-first), но центр держим безопасным.

- `bg_stage.jpg` (или png) — **1080×1920**, задник танцпола: тёмная сцена,
  колонки/лучи света, намёк на диджей-пульт. Центр (где поле) — спокойный,
  чтобы не спорил со стрелками
- `bg_crowd.png` — **1080×~400**, силуэты толпы прозрачным PNG по нижнему краю
  (для лёгкого «движения» можно 2–3 кадра: `bg_crowd_1..3.png`)
- `floor_glow.png` — **256×80**, мягкое световое пятно-подиум под танцором
  (сейчас CSS-градиент; картинкой будет богаче)
- (опц.) `bg_topbar.png` — подложка под HUD/прогресс, если нужен «экранный» вид

> Реактивность по комбо (`--energy`) сейчас на CSS-градиентах. С картинками
> оставим градиент сверху как «накал» поверх задника.

## 5. Эффекты 🟢

- `particle.png` — **32×32**, искра/блёстка (заменит кружки-партиклы при попадании)
  - можно 2–3 вариации: `particle_1..3.png`
- `spark_perfect.png` — **128×128**, вспышка на «Perfect» (звезда/блик)
- `beam_light.png` — **256×512**, луч прожектора (пульс под бит, опц.)
- `confetti.png` — **атлас 8–12 кадров**, конфетти на финал/большое комбо (опц.)

## 6. UI 🟢

Сейчас всё на CSS/Unicode. Под кастом:
- `btn_<dir>.png` ×4 — тач-кнопки направлений (up/down/left/right), **140×140**
  (×2). Норм-состояние; нажатое сделаю кодом (или ещё ×4 `btn_<dir>_active.png`)
- `icon_combo.png`, `icon_score.png` — мелкие иконки HUD (опц., **48×48**)
- `panel_modal.png` (или 9-slice) — подложка модалок, если нужен арт вместо CSS
- `logo.svg` — лого ивента/проекта для модалок

---

## 7. Звук 🟡

- `track.mp3` (+`.ogg` для совместимости) — основная музыка. Длину пропишем в
  `CONFIG.trackDuration`. **Дай BPM** — завяжу спавн стрелок «в такт»
- `sfx_hit.mp3` — обычное попадание (Good)
- `sfx_perfect.mp3` — точное попадание
- `sfx_miss.mp3` — промах
- `sfx_combo.mp3` — на круглых комбо (10/20/30…) (опц.)
- `sfx_finish.mp3` — финал/аплодисменты
- Все SFX: короткие (<1с), mp3 ~128–192 kbps, нормализованные по громкости

## Приоритетный «боевой» минимум (чтобы выглядело круто)

1. **Танцор** — 6 спрайт-листов (или хотя бы 6 поз) 🔴
2. **Стрелки** — 4 цветных PNG 🔴
3. **Фон** — `bg_stage` + `floor_glow` 🟡
4. **Звук** — `track.mp3` + `sfx_hit/perfect/miss` 🟡

Если жёсткий бюджет — закажи только **танцора** (6 поз/листов): остальное я
правдоподобно подделаю кодом. Живой персонаж — единственное, что кодом не нарисуешь.

## Общие технические правила

- Прозрачность: PNG-24 с альфой (или SVG/Lottie где векторно)
- Ретина: отдавать ×2 от логических размеров выше
- Цветовой профиль: sRGB
- Нейминг: строчные, `snake_case`, латиница, как в списках выше
- Спрайт-листы: кадры строго одинакового бокса, без обрезки по краям, отступ 10%
- Складывать в `assets/` (танцор → `assets/dancer/`, звук → `assets/sfx/`)
- Дай **спек-лист по кадрам** (сколько кадров в каждом атласе) — пропишу в конфиг
